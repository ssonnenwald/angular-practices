Factory Pattern

The Factory Pattern is a creational design pattern used to create objects without specifying the exact class of object that will be created.  In this pattern, a Factory method or class handles the object creation.

Implementation

In the case of Angular 19 with standalone components, let's create an example that demonstrates the Factory Pattern to instantiate different types of Vehicle objects (e.g., Car, Truck, Bike) based on some criteria.

  1. Create the Product Interface (Common Interface for all Products)
We define an interface for our vehicles to ensure that all of them have the same structure and can be used interchangeably.
  1. Create the Concrete Products (Classes Implementing the Vehicle Interface)
Here we define Car, Truck, and Bike as concrete implementations of the Vehicle interface.
  1. Create the Factory (To Produce the Products)
The Factory class will return the correct type of vehicle based on the input.
  1. Create the Standalone Component (To Use the Factory)
In this standalone component, we will use the factory to create a specific type of vehicle based on user input.

Explanation:

  • Vehicle Interface:  This defines the methods (startEngine and stopEngine) that any vehicle must implement.
  • Concrete Vehicles (Car, Truck, Bike):  Each vehicle implements the Vehicle interface and provides its own version of startEngine and stopEngine.
  • VehicleFactory:  The factory class decides which type of vehicle to create based on the input (e.g., car, truck, or bike).
  • AppComponent:  This component allows the user to select a vehicle type from a dropdown, and it uses the VehicleFactory to create the appropriate vehicle.  The vehicleDetails string is updated to show the actions (startEngine and stopEngine) of the created vehicle.

Conclusion:

This example demonstrates how to use the Factory Pattern in Angular 19 with standalone components to create different types of objects (vehicles) based on input.  The Factory Pattern abstracts the creation process and allows for easy modification or extension of the product types in the future.
