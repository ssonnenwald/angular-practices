Abstract Class

In Angular 19 with standalone components, you can use abstract classes to define base functionality that must be shared across multiple classes.  Abstract classes cannot be instantiated directly, but they can be extended by other classes, allowing for code reuse and providing a common structure for related components or services.

Conclusion:

This example demonstrates how to use an abstract class in Angular 19 with standalone components to create a flexible, extensible notification system.  The NotificationService abstract class enforces a common structure, and the concrete classes (EmailNotificationService and SMSNotificationService) implement specific logic for different types of notifications.

By using abstract classes, we can provide a clear contract for the subclass implementation, making it easy to add new types of notification services in the future.
