Key Features of This Structure

  1. Standalone Components
    • Each component is self-contained with its .ts, .html, and .scss files.
    • app.config.ts is used for bootstrap configuration instead of NgModule.
  2. Feature-Based Organization
    • Each feature (e.g., home, about) has its own directory.
    • Inside each feature folder, we separate components and views.
  3. Shared Resources
    • Shared components (buttons, modals) go under /shared/components/.
    • Reusable directives go in /shared/directives/.
    • Reusable pipes go in /shared/pipes/.
  4. Core Services & Configuration
    • Authentication, guards, and interceptors are inside /core/.
    • Global configurations like app-wide settings are in /core/config/.
  5. Layouts for UI Consistency
    • Global layout components (header, footer, sidebar) are in /layout/.
