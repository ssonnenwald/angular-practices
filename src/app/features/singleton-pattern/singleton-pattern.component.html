Singleton Pattern

Singleton is a creational design pattern, which ensures that only one object of its kind exists and provides a single point of access to it for any other code.

  1. Creating the Singleton Service
In Angular, the service can still be provided at the root level to ensure that it follows the Singleton pattern.

Key Points:

  • providedIn: 'root' ensures that the service is a singleton throughout the application (Angular will create it only once).
  • The getInstance() method ensures that only one instance of the service is ever created.
  1. Using the Singleton Service in a Component
To use the singleton service in a component, you simply inject the service as usual. Angular ensures it’s the same instance throughout the application.

Explanation:

  • In the service, getInstance() controls the creation of the singleton instance. If the instance doesn’t exist, it is created, otherwise the existing one is returned.
  • The component injects the SingletonService, and you can use its getData() method or any other methods you define.
