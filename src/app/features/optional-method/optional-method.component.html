Optional Method

In TypeScript (and Angular 19 with standalone components), you can define optional methods by using optional properties in interfaces or classes.  This is useful when you want a method or property to be available but not necessarily implemented by every class or component.

Implementation

Let's say you have a system where you have different types of notification services (e.g., Email, SMS, and Push).  Some of these services might implement an optional method, such as trackDeliveryStatus(), while others may not.

We can define the optional method in an interface and implement it conditionally in certain services.

Explanation:

  • Optional Method in Interface:  In the NotificationService interface, trackDeliveryStatus?() is an optional method.  Classes implementing this interface can choose to implement it or not.  This makes the method flexible and non-mandatory for all services.
  • Concrete Classes:  EmailNotificationService implements the trackDeliveryStatus() method, but SMSNotificationService does not.  This shows how you can have an optional method that doesn’t always need to be implemented.
  • Component Logic:  The AppComponent provides functionality to switch between the Email and SMS notification services.  It also includes a Track Delivery Status button that checks if the service has implemented the trackDeliveryStatus() method.  If the method is available, it is called; otherwise, a default message is shown.

Conclusion:

This example demonstrates how to use optional methods in Angular 19 with standalone components.  By marking methods as optional in an interface, you can provide flexibility in how services or components implement certain functionality.  This approach allows you to create more adaptable and modular code.
